# Security Policy

## Supported Versions

Security updates are currently provided for the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security reports.

1. Email the maintainer at `security@invalid.local` with:
   - A clear description of the issue
   - Steps to reproduce
   - Potential impact
2. Use the subject line: `Email-Scraper Security Report`.
3. Expect an initial response within 72 hours.

If the issue is confirmed, a fix will be developed and released as quickly as possible. Public disclosure should wait until a patched release is available.

## Security Best Practices for Users

- Do not commit API keys into source control.
- Prefer environment variables over CLI secrets when running in shared environments.
- Respect robots.txt, local regulations, and applicable outreach laws (for example CAN-SPAM and GDPR).
